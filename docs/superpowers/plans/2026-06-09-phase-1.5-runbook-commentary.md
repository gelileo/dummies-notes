# Phase 1.5 — runbook + commentary — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every `concept-illustrator` figure, per frame, a **runbook** (build-spec that drives generation and is the ground truth for drift review) and a **commentary** (accessible, simple-sentence narration for slides/video), stored as required `figure.json` fields.

**Architecture:** Extend the shipped skill at `.claude/skills/concept-illustrator/`. `figure.json` `frames[]` gains required non-empty `runbook` and `commentary` strings alongside `file` and `caption`. The storyboard step becomes runbook-first (write runbook → draw SVG → write caption + commentary). `validate_figure` enforces the new fields. Docs (figure-json, voice-and-metaphor, review-protocol, SKILL.md) and the golden quicksort example are updated. The HTML viewer is unchanged (still shows only `caption`).

**Tech Stack:** Python 3 stdlib + unittest (as in Phase 1). No new dependencies. Spec: `docs/superpowers/specs/2026-06-09-runbook-commentary-design.md`.

**Conventions:**
- Run tests: `python3 -m unittest discover -s .claude/skills/concept-illustrator/scripts/tests -p 'test_*.py'`
- **Living-doc drift gate.** `drift-check` (pre-commit) maps changed code paths to `knowledge/` articles via (a) each article's `affects:` frontmatter globs and (b) the CLAUDE.md mapping table; a changed file that matches must be accompanied by its article's update in the same commit. Today `illustration-engine.md` maps the whole skill dir (`.claude/skills/concept-illustrator/**`). **Task 1 narrows that** to the contract + tool (`SKILL.md`, `scripts/render.py`), so reference-doc, example, and test-file edits no longer force an article touch. After Task 1: tasks that change `SKILL.md` or `scripts/render.py` MUST update `illustration-engine.md` in the same commit; tasks that change only `references/**`, `examples/**`, or the test file need not. If a commit is unexpectedly blocked, STOP and report rather than using `--no-verify`.

## Commentary

Commentary is the per-frame narration in `figure.json` (`frames[].commentary`). It
is longer than the one-line `caption` — a short narration paragraph — meant to be
read aloud for slides and video. It is not shown in the HTML viewer.

- **Prefer simple sentences.** One idea per sentence. Avoid nested clauses and
  compound-complex sentences. Short sentences, plain words.
- **Vibrant through metaphor and rhythm, not length.** Engagement comes from a
  vivid image and good cadence, never from a long sentence.
- **No unexplained jargon.** An obscure term is a prerequisite to illustrate, not
  an aside to drop.
- **Faithful to the frame.** Narrate what this frame actually shows.
- **Transcript-ready.** It should read naturally when spoken.

Example (a quicksort partition frame):

> Now the real work starts. We walk left to right. The pivot is our yardstick.
> Every value smaller than it slides into a growing zone on the left.

Add a `## Runbook drift` section:

```markdown
## Runbook drift

Each frame's `runbook` (in `figure.json`) is the ground truth of intent. The
fidelity critic diffs the rendered SVG against its runbook: cell count, colour
roles, pointer positions, what changed from the previous frame. A mismatch is
drift. Two repair branches:

- **SVG is wrong** → regenerate that frame from its runbook.
- **Runbook is wrong or outdated** (the concept changed) → fix the runbook, then
  re-run; regeneration redraws from it.
```

In `## Blind-reader test`, add a sentence: the blind-reader sees ONLY the rendered figure (and at most the terse caption) — never the runbook or commentary; its read is compared against the `commentary`'s intent, and divergence is a comprehension gap.
