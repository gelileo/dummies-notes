RSA encryption

A way to send secret messages using two matching keys: a public key anyone can use to lock a message, and a private key that only the owner keeps and can use to unlock it.

Builds on:

  • Modular arithmetic — RSA locks and unlocks a message by doing wrap-around calculations, so you cannot follow how it works without first seeing how counting that wraps around behaves.
  • Prime numbers — RSA builds each pair of keys by secretly multiplying two large prime numbers, and its safety rests on how hard it is to find those primes again, so the reader must first know what a prime number is.
  • Asymmetric cryptography — RSA is one specific recipe for this two-key idea, so the reader needs the general two-key picture before the particular way RSA fills it in makes sense.
